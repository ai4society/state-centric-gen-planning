# Data Directory Documentation

This directory contains the complete lifecycle of planning data, from raw PDDL to trained model checkpoints.

## Directory Structure

```text
data/
├── pddl/                  # Raw Domain and Problem definitions
│   ├── <domain>/          # e.g., blocks, gripper, etc.
│   │   ├── domain.pddl
│   │   ├── train/
│   │   ├── validation/
│   │   ├── test-interpolation/
│   │   └── test-extrapolation/
├── plans/                 # SAS+ Plans generated by Fast Downward
├── states/                # State Trajectories (sequences of predicates)
└── encodings/             # ML-Ready Vector Representations
    └── graphs/            # Fixed-size WL Graph Embeddings
        ├── <domain>/
        │   └── <split>/
        │       ├── <problem>.npy       # Trajectory matrix [T, D]
        │       └── <problem>_goal.npy  # Goal vector [D]
```

## Output Directories

When running `2_unified_train_eval.slurm` from the root dir., artifacts are generated in the root directory (not inside `data/`), organized by **Encoding**, **Model**, and **Mode**.

### Checkpoints (`checkpoints/`)

Contains trained model weights (`.pt` for PyTorch, `.json` for XGBoost).

```text
checkpoints/
└── graphs/                        # Encoding Type
    ├── lstm_delta/                # Model + Mode
    │   ├── blocks_lstm_best.pt
    │   └── ...
    ├── xgboost_delta/
    │   ├── blocks_xgb.json
    │   └── ...
    └── ...
```

### Results (`results/`)

Contains inference logs and JSON summaries (Solved rates, Plan lengths).

```text
results/
└── graphs/
    ├── lstm_delta/
    │   ├── blocks_validation_delta_results.json
    │   ├── blocks_test-extrapolation_delta_results.json
    │   └── ...
    └── xgboost_delta/
        └── ...
```

## Data Formats

### 1. Graph Embeddings (`.npy`)

Generated by `wlplan`.

- **Trajectory**: A float32 matrix of shape `(T, D)`, where $T$ is time steps and $D$ is the vocabulary size of the WL kernel.
- **Goal**: A float32 vector of shape `(D,)`.

### 2. XGBoost Flattening

For XGBoost, the `dataset.py` utility flattens the sequential `.npy` data on the fly:

- **Input**: Concatenation of Current State and Goal. Shape: `(1, 2*D)`.
- **Target**: Next State (or Delta). Shape: `(1, D)`.
