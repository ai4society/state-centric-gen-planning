# Dataset & Encodings

This directory manages the lifecycle of planning data, from raw PDDL to machine-learning-ready vectors.

## Directory Hierarchy

```text
data/
├── pddl/                  # Raw Domain/Problem definitions
│   ├── <domain>/          # e.g., blocks, gripper
│   │   ├── train/         # Small problems (2-5 objects)
│   │   ├── validation/    # Unseen small problems
│   │   ├── test-interp/   # Medium problems
│   │   └── test-extrap/   # Large OOD problems (10+ objects)
├── plans/                 # SAS+ Plans generated by Fast Downward
├── states/                # State Trajectories (sequences of predicates)
└── encodings/             # Vector Representations
    ├── graphs/            # Weisfeiler-Leman (WL) Graph Embeddings
    └── fsf/               # Fixed-Size Factored Embeddings
```

## PDDL Data (`data/pddl/`)

This directory contains the symbolic definitions of the planning tasks. There are taken from the [Symmetry-Aware Transformer Training for Automated Planning Repository](https://github.com/mrlab-ai/symmetry-transformers/tree/main) ([permalink when copied](https://github.com/mrlab-ai/symmetry-transformers/tree/56983a26a37e28e2f148ce067565c9d8eefb85eb/pddl)). Some processing was done, such as lowercasing. The splits are as follows:

- `train`: Small instances (e.g., 2-5 blocks) used for learning.
- `validation`: Slightly larger instances for hyperparameter tuning.
- `test-interpolation`: Instances of similar size to training but unseen configurations.
- `test-extrapolation`: Large instances (e.g., 10+ blocks) to test Out-of-Distribution (OOD) generalization.

## Encoding Strategies

We compare two distinct state representations:

### 1. Graph Embeddings (`encodings/graphs/`)

- **Method:** Weisfeiler-Leman (WL) Algorithm.
- **Logic:** Converts the PDDL state into a graph (Objects=Nodes, Predicates=Edges). We hash the graph to extract structural features.
- **Pros:** Domain-agnostic, handles varying object counts naturally.
- **Format:** `.npy` files containing sparse histograms of WL colors.

### 2. Fixed-Size Factored (`encodings/fsf/`)

- **Method:** Position-based vectors.
- **Logic:** Maps objects to fixed slots in a vector (size = Max Objects in domain).
- **Cons:** Fails to generalize if $N_{test} > N_{train}$. Used to demonstrate the necessity of graph representations.
- **Format:** `.npy` files containing dense vectors.

## File Formats

**Trajectory Files (`.npy`):**

- Shape: `(T, D)` where $T$ is plan length and $D$ is embedding dimension.
- Contains the sequence of states $[S_0, S_1, \dots, S_T]$.

**Goal Files (`_goal.npy`):**

- Shape: `(D,)`.
- Contains the embedding of the goal condition.
